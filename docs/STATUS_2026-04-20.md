## Live pipeline (what runs on every setup)

1. `candles_ok` — 1H/4H/1D/15m all fetched
2. `formation_found` — M/W or fallback scalp detected
3. `htf_aligned` — **4H trend hard-veto** (counter-trend rejected unless exempt reversal variant + trend not accelerating)
4. `level_ok` — post-formation Level 1/2/3 analysis
5. `phase_valid` — inside entry-eligible weekly phase
6. `direction_ok` — weekly bias + FMWB real-move check
7. `target_acquired` — TP tiers computed (with 3-way collapse guard → R-multiple fallback)
8. `rr_passed` — R:R ≥ min
9. `scored` — confluence computed (direction-aware, multi-session bonus, ADR 1.0% tolerance)
10. `confluence_passed` — score ≥ `min_confluence` (default 35%)
11. `retest_passed` — 2+ of 4 retest conditions
12. `sanity_agent_passed` — **Opus 4.7 LLM veto**
13. `signal_built` → sizing (with notional cap + aggregate-risk budget) → entry

## Known open items

1. **Cooldown not persisted** — `_cooldowns` dict is in-memory only. Wipes on deploy. Noticed 2026-04-20 when BNB re-entered 22 min after close (deploy cycle). Workaround: the 2h scratch rule catches most rapid re-entries.
2. **Correlation provider is live-only** — yfinance historical fetch is current-market-day only. Replay hit-rate shows 0%; live firing works. Documented, not a bug.
3. **Agent `trade_id` back-link** — after a trade enters, `mm_agent_decisions.trade_id` is not updated. Audit tool uses timestamp match as workaround. 20-line fix, not urgent.
4. **NYC reversal not a named detector** — works via generic session-changeover classification, not a dedicated detector.
5. **Trailing stop warning** — course explicitly forbids trailing stops; engine doesn't warn if one is configured externally on the exchange (live-mode only concern).

None of these is blocking. All non-destructive.

## Recommended workflow going forward

1. **When a live trade does something unexpected:**
   1. `python3 scripts/trade_audit.py <trade_id>` — read everything at a glance
   2. If the cause is unclear: `python3 scripts/replay_scan.py --symbol X --days 30 --factor-rates`
   3. **Re-read the course before changing any rule** (anti-pattern #3 from CHANGELOG)

2. **When proposing a rule change:**
   1. Test the hypothesis with `replay_scan.py --min-confluence N` (or whichever flag)
   2. Cite the exact course lesson + timestamp in the commit message
   3. Update CHANGELOG.md with the course quote

3. **When the agent returns something surprising:**
   1. Query `mm_agent_decisions` for the full reasoning
   2. If the agent was wrong, add a fixture to `tests/fixtures/mm_sanity/` so it regression-guards
