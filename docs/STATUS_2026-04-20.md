## What's live in v38

### Deterministic pipeline (12 gates, in order)

1. `candles_ok` — 1H/4H/1D/15m all fetched
2. `formation_found` — M/W or fallback scalp detected
3. `htf_aligned` — **NEW** — 4H trend veto
4. `level_ok` — post-formation Level 1/2/3 analysis
5. `phase_valid` — inside entry-eligible weekly phase
6. `direction_ok` — weekly bias + FMWB real-move check
7. `target_acquired` — TP tiers computed (with new 3-way collapse guard)
8. `rr_passed` — R:R ≥ min
9. `scored` — confluence computed (direction-aware, multi-session bonus)
10. `confluence_passed` — score ≥ `min_confluence` (default 35%)
11. `retest_passed` — 2+ of 4 retest conditions
12. `sanity_agent_passed` — **NEW** — Opus 4.7 veto layer
13. `signal_built` → sizing → entry

### Agent 4 (MM Sanity Agent)

- Model: `claude-opus-4-7` with adaptive thinking, `effort=high`
- Prompt: system prompt cached 1h TTL, rubric + 3 worked fixtures
- Fail-open on API error / timeout / missing SDK
- Budget cap: $600/mo, auto-downgrade to Sonnet 4.6 at 90%
- Per-decision cost: ~$0.05 typical, latency ~3s
- Kill switch: `MM_SANITY_AGENT_ENABLED=false`

### Risk protection stack

| Layer | Limit | Enforcement |
|---|---|---|
| Per-trade dollar risk | 1% of account | Hard (via position sizing) |
| Per-trade notional | 50% of account | Hard (via `mm_max_notional_pct`) |
| Aggregate open risk | 5% of account | Hard (via `mm_max_aggregate_risk_pct`) |
| Position count (backstop) | 20 | Hard (raised from 3) |
| Counter-trend into accelerating HTF | Rejected | Hard (HTF veto) |
| SL distance > 5% | Warning | Soft — logged only |
| Grade F | Allowed if ≥ 35% confluence | Soft — agent reviews |
