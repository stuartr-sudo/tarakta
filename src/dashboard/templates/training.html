01 System Overview ▶

Tarakta is an automated cryptocurrency futures trading bot that combines Smart Money Concepts (institutional trading strategies) with AI-powered decision making. It trades perpetual futures on Binance using leverage.

Core idea: The bot watches crypto markets around the clock, identifies trading opportunities that institutional traders would look for, then uses three AI agents to decide whether and when to trade. It can go both long (profit when price rises) and short (profit when price falls).

The Full Pipeline

Scanner (every N min) → Confluence scoring → Consensus check → Agent 1 analyses → Entry Refiner queues → Footprint gate → Agent 2 times entry → Order executes → Agent 3 manages → Lesson generated

How a Scan Cycle Works

Every scan interval (configurable, typically every few minutes), the engine:

  • Fetches 1H, 4H, and 1D candle data for all configured trading pairs
  • Runs market structure analysis across all timeframes to determine trend direction
  • Checks session timing (Asian, London, New York kill zones)
  • Runs sweep detection on the 1H timeframe — a completed liquidity sweep is the primary entry trigger
  • Confirms displacement (strong volume move away from the sweep level)
  • Scores the signal using the PostSweepEngine confluence formula
  • If score passes the entry threshold, enriches the signal with structural data (order blocks, FVGs, liquidity levels) and sends it to Agent 1

Scanner Mode: Expanded-Footprint

The active scanner mode strips out retail-level indicators (standalone OBs, FVGs, premium/discount zones) and focuses purely on sweep-first setups. A completed liquidity sweep on the 1H timeframe is the only entry trigger. OBs, FVGs, and liquidity levels are still computed but used only as context for the AI agents, not for scoring.

Instance Isolation

Multiple bot instances can run simultaneously against the same exchange and database. Each instance has a unique INSTANCE_ID that keeps its engine state, open positions, and configuration overrides separate. This allows A/B testing of different strategies or settings.

Infrastructure

  • Hosting: Fly.io, Amsterdam region (shared-cpu-1x, 2GB RAM)
  • Exchange: Binance Futures (perpetual contracts)
  • Database: Supabase (PostgreSQL with pgvector extension)
  • AI: OpenAI GPT models for agents, OpenAI embeddings for RAG
  • Dashboard: FastAPI + Jinja2 web interface on port 8080
02 The Three AI Agents (Deep Dive) ▶

Each agent is an OpenAI GPT model call with a detailed system prompt, structured JSON output, and rich context injected into the user prompt. They operate as a sequential chain — each agent's output feeds the next.

Agent 1 — Strategic Entry Analyst

Answers: "Is this trade worth taking, and in which direction?"

Agent 1 is the direction authority. It independently decides whether to go LONG or SHORT regardless of the scanner's initial suggestion. It returns one of three verdicts:

  • SETUP_CONFIRMED: Enter now — the signal is strong enough for immediate action. Agent 2 gets called on the very next tick.
  • WAIT_PULLBACK: The thesis is valid but the price is not yet at an optimal entry. The signal is queued in the Entry Refiner, and Agent 2 checks every 5 minutes for a pullback into the entry zone.
  • SKIP: Reject this signal entirely.

What Agent 1 Actually Sees

Agent 1 receives an extensive context package including:

  • Sweep details (level, depth, type)
  • HTF market structure (1H/4H/1D trends)
  • 5 nearest order blocks with distance %
  • 5 nearest fair value gaps with midpoints
  • Liquidity pools (buy & sell side)
  • Relative volume (RVOL multiplier)
  • Displacement detection & strength
  • Pullback status & retracement %
  • Session timing (kill zone status)
  • Fibonacci retracement levels (OTE zone)
  • Leverage profile (funding, OI, L/S ratio)
  • Judas swing probability
  • Liquidation distances (long & short)
  • BTC trend & 24h change
  • Sentiment score (-10 to +10)
  • Recent news headlines
  • Last 5 trades on this symbol (outcome, PnL)
  • RAG knowledge (similar past trades)
  • Learned lessons from post-mortems
  • Advisor insights (missed opportunities)
  • Bot performance metrics (win rate, streaks)
  • Algorithmic SL/TP reference levels

Agent 1 outputs: action, direction (LONG/SHORT), confidence score, detailed reasoning, suggested entry zone, SL, TP, invalidation level, must-reach price, and market regime classification.

Agent 2 — Tactical Entry Timer

Answers: "Is NOW the right moment to enter?"

Agent 2 is called every 5 minutes while a signal is queued. It receives Agent 1's locked direction and thesis (cannot change it) plus fresh real-time data. It performs a mandatory 4-step analysis:

  • Step 0 — Must-reach-price gate: Has the institutional displacement move occurred? If not, WAIT.
  • Step 1 — Rejection check: Has price tested the entry zone and shown a rejection candle (wick ratio, engulfing pattern, volume spike)?
  • Step 2 — Live vs continuation: Is this a live rejection or stale data?
  • Step 3 — Structural confirmation: Do order blocks, FVGs, and market structure support entry here?

What Agent 2 Sees (beyond Agent 1's context)

  • Last 50 five-minute candles (OHLCV table)
  • Order book (top 10 bid/ask, walls, imbalance)
  • Footprint / order flow (delta %, absorption)
  • Cumulative delta confirmation
  • OI change percentage
  • Buy/sell volume split
  • Previous Agent 2 analysis (reasoning continuity)
  • Check count (how many times evaluated)
  • Price change since queued
  • Sentiment score & headlines
  • Advisor insights

Agent 2 outputs: ENTER (with exact entry price, SL, TP, position size modifier 0.25–1.5x, confidence, urgency) or WAIT.

Agent 3 — Position Manager

Answers: "Should I adjust this open trade?"

Agent 3 monitors every open position and can do exactly two things:

  • TIGHTEN_SL: Move stop-loss closer to current price (never wider). Requires confidence ≥ 50.
  • EXTEND_TP3: Push the final take-profit target further. Requires confidence ≥ 60 and evidence of strong momentum (BOS, displacement, no resistance ahead).

Agent 3 defaults to HOLD in 80%+ of cases. It cannot close trades, widen stops, or override the exchange's hard SL/TP levels.

What Agent 3 Sees

  • Position details (entry, SL, TP, direction)
  • Current PnL ($ and %)
  • Unrealized R ratio
  • TP tier progress (X/3 filled)
  • Time held
  • Remaining position quantity
  • 1H ATR (volatility baseline)
  • Agent 1's original thesis
  • BTC trend + funding rate
  • HTF market structure per timeframe
  • Last 15-20 candles (5m + 1h)
  • Position management lessons

Temperature Settings

Temperature controls how random or deterministic the AI's output is. Lower = more consistent decisions. Agent 1: 0.4 (strategic but stable), Agent 2: 0.5 (some flexibility for timing), Agent 3: 0.3 (very deterministic, safety-critical).
03 Signal Scanning & Confluence Scoring ▶

The scanner checks all configured markets on a timer. In expanded-footprint mode, the only entry trigger is a completed liquidity sweep on the 1H timeframe, confirmed by displacement volume.

Confluence Score Formula

Each signal is scored out of 100 base points (plus up to 10 bonus points for leverage alignment). The score components are:

ComponentPointsWhat It Checks
Sweep Detected35 ptsA completed liquidity sweep on 1H — price spiked past a key level, triggered stops, and reversed. This is the primary signal.
Displacement Confirmed25 ptsStrong volume move away from the sweep level. Proves institutional commitment behind the reversal.
HTF Direction Aligned15 ptsThe 4H and/or 1D market structure trend agrees with the trade direction.
Post-Kill-Zone Timing15 ptsThe signal occurs shortly after an active session kill zone (London or New York open), when institutional activity is highest.
Pullback Confirmed10 ptsPrice has pulled back into the optimal trade entry (OTE) zone — the 61.8%–78.6% Fibonacci retracement of the displacement move.
Leverage Aligned (bonus)0–10 ptsFunding rate, open interest, and long/short ratio all support the direction. Only on futures.
Entry Threshold: Signals must reach the configured minimum score (set in Settings) to be passed to Agent 1. A typical threshold is 60–70. A score of 100 means every base component aligned perfectly.

Weekly Cycle Adjustments

The scanner applies timing adjustments based on the day of the week:

  • Monday manipulation window: Score penalty during early Monday hours when institutions often create fake moves to trap retail traders.
  • Mid-week reversal bonus: Small score boost during Tuesday–Thursday when genuine trend moves are more likely.

Consensus Check

Before reaching Agent 1, signals pass through a consensus mechanism that checks directional alignment:

  • Portfolio bias: If most open profitable positions are in the opposite direction, a penalty of −10 points is applied.
  • BTC trend disagreement: If BTC's 4H/1D structure contradicts the signal direction, an additional −5 points is applied.

If a signal's score drops below the threshold after penalties, it is queued in the Consensus Monitor and re-evaluated every 60 seconds for up to 30 minutes. If alignment improves, the signal graduates (possibly with a direction flip).

Session Timing & Kill Zones

Crypto markets run 24/7, but institutional activity follows three major trading sessions:

  • Asian session (00:00–08:00 UTC): Typically lower volume. Price often establishes a range.
  • London session (07:00–16:00 UTC): High volume. Institutions often sweep Asian session highs/lows.
  • New York session (12:00–21:00 UTC): Highest volume. The most significant moves typically occur here.

The "kill zone" is the first 1–2 hours of each session when manipulation is most likely. The scanner gives a 15-point bonus for signals detected in the post-kill-zone period, when genuine reversals tend to follow the initial sweep.

Footprint Gate (Order Flow Confirmation)

After Agent 2 says ENTER, there is one final gate: the Footprint Analyzer examines real-time order flow data (trade tape, open interest changes) to confirm that actual buyer/seller pressure supports the direction.

  • Delta %: The ratio of aggressive buys to sells. For a long trade, delta should be positive.
  • Absorption: Large buy orders sitting at a level and absorbing sell pressure (for longs). Indicates institutional support.
  • Cumulative delta: The running total of buy minus sell volume confirming directional commitment.
Note: Footprint data is ephemeral — it comes from the real-time trade tape and cannot be fetched historically. If the footprint gate fails, the trade is rejected even if Agent 2 said ENTER.
04 Risk Management & Safety Systems ▶

Risk management is the most important part of the system. These controls exist to protect capital even when individual trades go wrong or market conditions become extreme.

Per-Trade Risk Controls

ControlWhat It Does
Max Risk %Limits the maximum loss on any single trade as a percentage of balance. If set to 2% on a $10,000 balance, no trade can lose more than $200.
Max SL %Caps how far the stop-loss can be from the entry price. Prevents the agents from setting overly wide stops that expose too much capital.
Min R:R RatioMinimum reward-to-risk ratio required. If set to 2.0, the potential profit must be at least 2x the potential loss before a trade is placed.
Max Position %Maximum percentage of balance used as margin for a single position.
Max Position Volume %Caps position size relative to the symbol's 24h trading volume, preventing market impact on low-liquidity pairs.

Portfolio-Level Controls

ControlWhat It Does
Max Concurrent PositionsLimits how many trades can be open simultaneously. Prevents overexposure.
Max Exposure %Total portfolio exposure limit across all open positions combined.
Max Per Correlation ClusterLimits how many correlated positions can be open (e.g., only N altcoin longs at once).
Max Daily TradesHard cap on trades per day, preventing overtrading.

Circuit Breaker

Emergency brake. Automatically halts ALL trading if total losses exceed a threshold (e.g., 15% of balance). When triggered, no new trades are placed until the engine is manually reset. This prevents catastrophic losses during black swan events or strategy failure.

Daily Drawdown Limit

If the bot loses more than the configured daily limit, it pauses trading until the next calendar day. This prevents compounding losses during bad market conditions.

Cooldown Period

After consecutive losses, the bot enters a configurable cooldown where no new trades are placed. This prevents emotional "tilt" trading — chasing losses with increasingly desperate entries.

BTC Macro Gate

When enabled, the bot checks Bitcoin's trend before entering altcoin trades. If BTC is crashing (e.g., down >2% in an hour), altcoin longs are blocked because altcoins tend to follow BTC downward regardless of their individual setups.

Sentiment Gate

Extreme negative sentiment (measured on a -10 to +10 scale from news and social data) can block trades. If sentiment is strongly bearish and the signal is long, the trade may be rejected or its score reduced.

Funding Rate Gate

On futures, the funding rate indicates market positioning. Extremely positive funding means the market is crowded long (risky for longs), and vice versa. When enabled, this gate blocks trades that go against heavily funded positions.

05 Position Management & Trade Lifecycle ▶

Full Trade Lifecycle

Signal scored → Agent 1 approves → Queued in Entry Refiner → Agent 2 says ENTER → Footprint gate passes → Order placed on exchange → Agent 3 monitors → Trade closes (SL/TP/time) → Lesson generated

Entry Refiner & Pullback Queue

When Agent 1 says WAIT_PULLBACK, the signal enters the Entry Refiner. This is a queue that holds signals and waits for price to pull back into the optimal entry zone:

  • Entry zone: Defined by Agent 1 (zone_high to zone_low), typically based on a structural level like an order block, FVG, or Fibonacci retracement zone.
  • OTE zone: The Optimal Trade Entry zone is the 61.8%–78.6% Fibonacci retracement of the displacement move. This is where institutional buyers/sellers are most likely to re-enter.
  • Must-reach price: A price level that must be hit before pullback entry is valid. Ensures the displacement move has actually occurred.
  • Agent 2 checks: Every 5 minutes, the entry refiner fetches fresh 5m candles, order book data, and footprint data, then asks Agent 2 whether conditions are right to enter.
  • Expiry: Signals expire after a configurable period if Agent 2 never says ENTER. The entry zone can be updated up to a few times if price action warrants it.

Take-Profit Tiers

Each trade has three take-profit targets, which close portions of the position progressively:

TierPurposeBehaviour
TP1Quick lock-inCloses a portion of the position early. Reduces risk by banking some profit immediately.
TP2Medium targetCloses another portion at a higher profit target. By now, the stop-loss is typically moved to breakeven or into profit.
TP3Full target / runnerThe remaining position. Can be extended by Agent 3 if momentum is strong. This is where the biggest profits come from.

Trailing Stop & Breakeven

  • Breakeven activation: After a trade reaches a configurable R:R ratio (e.g., 1.5R), the stop-loss is automatically moved to breakeven (entry price). From this point, the trade cannot lose money.
  • Trailing activation: After reaching a higher R:R threshold, a trailing stop activates that follows price, locking in progressively more profit.

How Trades Close

  • Stop-loss hit: Trade closes at a loss (or breakeven/profit if SL was tightened).
  • All TP tiers filled: Full profit captured across all three levels.
  • Max hold time exceeded: Configurable time limit. Prevents capital from being tied up in stale trades.
  • Stale close below R:R: If a trade is held for too long with low R:R, it can be closed as a "stale" position.

Post-Trade: Lesson Generation

After every trade closes, the Lesson Generator analyses the outcome. It sees all three agents' reasoning, the signal components, and the trade result. It produces structured lessons tagged with severity (low/medium/high/critical), lesson type (entry/timing/exit/position_management/risk), and which agents should learn from it. These lessons are stored in the database and injected into future agent prompts.

06 AI Learning & Knowledge Systems ▶

Tarakta has three systems that make agents smarter over time: the RAG knowledge base, the lesson generator, and the trade advisor.

RAG (Retrieval-Augmented Generation)

Purpose: Before making a decision, agents can search past trade history for similar setups and see how those trades turned out.

How it works:

  • Ingestion: Every closed trade is converted into a text summary (symbol, direction, outcome, PnL, confluence score, signal reasons, agent reasoning) and embedded as a 1536-dimensional vector using OpenAI's text-embedding-3-small model.
  • Storage: Embeddings are stored in Supabase using the pgvector extension alongside the text content in the knowledge_chunks table.
  • Retrieval: When a new signal arrives, a query is built from the symbol, direction, sweep type, market regime, and displacement status. This query is embedded and compared against all stored trades using hybrid search.
  • Hybrid search: Combines dense search (cosine similarity between vectors) with lexical search (PostgreSQL full-text search) using Reciprocal Rank Fusion (RRF) to rank results.
  • Injection: The top 5 most relevant past trades are formatted and injected into Agent 1 and Agent 2's context as "Similar Past Trades" with relevance scores.

Example: If the bot is evaluating a LONG setup on ETH after a sweep of a Fibonacci level, the RAG system might surface past ETH LONG trades with similar sweep types and show their outcomes, helping the agent avoid repeated mistakes.

Lesson Generator

Purpose: Extract actionable rules from every closed trade so agents learn from mistakes and successes.

After each trade closes, the lesson generator:

  • Reviews all three agents' reasoning, the trade outcome, and the signal quality
  • Generates structured lessons in "IF [condition], THEN [action]" format
  • Tags each lesson with: severity (low/medium/high/critical), type (entry/timing/exit/position_management/risk), and applies_to (agent1/agent2/agent3)
  • Avoids duplicating lessons already generated (checks last 5 existing lessons)

Lessons are stored in the trade_lessons table and injected into the relevant agent's prompt with priority ordering: symbol-specific agent lessons first, then high-severity general lessons, then medium severity.

Trade Advisor

Purpose: Identify missed opportunities and patterns the bot is consistently missing.

The advisor runs periodically (daily on reset or manually via the dashboard) and:

  • Fetches high-scoring signals that were never traded (acted_on = false)
  • Simulates what would have happened if the trade had been taken (2% SL, 4% TP, max 4h hold)
  • Uses Claude (Anthropic's AI) to analyse patterns across the missed signals
  • Produces recommendations (e.g., "lower entry threshold for BTC sweeps" or "the bot is too conservative during NY session")
  • Stores insights in the advisor_insights table

These insights are formatted and injected into Agent 1 and Agent 2's context so they can factor in the advisor's recommendations.

07 Smart Money Concepts Explained ▶

Smart Money Concepts (SMC) is a trading methodology based on understanding how large institutional traders (banks, hedge funds, market makers) operate. These institutions control the majority of market volume and their activity creates predictable patterns.

Liquidity Sweeps (Primary Signal)

The core thesis: Institutional traders need liquidity (orders to trade against) to fill their large positions. They deliberately push price to levels where retail traders have placed stop-losses, trigger those stops (creating liquidity), then reverse the market in their intended direction.

A liquidity sweep occurs when price briefly spikes above resistance or below support, triggers stop-losses, then reverses. In Tarakta's expanded-footprint mode, a confirmed sweep on the 1H timeframe is the ONLY trigger for a new signal. The bot looks for sweeps of various levels including swing highs/lows, Fibonacci levels, and equal highs/lows.

Displacement

After a sweep, institutions make a strong move in their intended direction. This "displacement" is characterised by large-body candles with high volume and minimal wicks. The scanner awards 25 points for confirmed displacement because it proves institutional commitment — the sweep was not random.

Fair Value Gaps (FVGs)

When price moves so fast that candle bodies don't overlap (candle 1's close and candle 3's open leave a gap with candle 2), it creates an imbalance zone. Price tends to return to fill these gaps before continuing. FVGs are used as potential entry zones — if the displacement creates an FVG, the pullback entry often targets the gap's midpoint.

Order Blocks (OBs)

An order block is the last candle before a strong institutional move. It represents the zone where institutions placed their orders. When price returns to an OB, it often finds support (bullish OB) or resistance (bearish OB). Agents use OBs to validate entry zones and set stop-losses.

Market Structure

Market structure describes the trend through swing highs and swing lows:

  • Break of Structure (BOS): Price breaks a recent swing high (bullish) or swing low (bearish), confirming trend continuation.
  • Change of Character (CHoCH): Price breaks structure in the opposite direction, signalling a potential trend reversal.

The bot analyses structure on three timeframes (1H, 4H, 1D) with strength percentages. Higher-timeframe alignment (HTF) adds 15 points to the confluence score.

Optimal Trade Entry (OTE)

The OTE zone is the 61.8%–78.6% Fibonacci retracement of a displacement move. This is the statistically optimal area where price tends to pull back before continuing. When Agent 1 says WAIT_PULLBACK, it typically defines the entry zone around the OTE.

Judas Swing

A fake move at the open of a trading session designed to trap traders on the wrong side before the real move. The bot calculates the probability of a Judas swing based on funding rates, open interest, and the crowded side of the market.

08 Leverage, Futures & Margin ▶

Tarakta trades perpetual futures contracts on Binance. Understanding leverage and margin is critical.

What Are Perpetual Futures?

Unlike spot trading (buying and holding crypto), futures contracts let you profit from both rising and falling prices:

  • Long: You profit when price goes up. You lose when price goes down.
  • Short: You profit when price goes down. You lose when price goes up.

"Perpetual" means the contract never expires. You can hold the position as long as you maintain margin.

Leverage Explained

Leverage amplifies your position size. At 10x leverage, $100 of margin controls $1,000 worth of a position. If price moves 1% in your favour, you make 10% on your margin. If it moves 1% against you, you lose 10%.

Higher leverage means:

  • Larger potential profits per trade
  • Larger potential losses per trade
  • Liquidation happens at a closer price level
  • Less margin tied up per position (more capital available for other trades)

Margin & Liquidation

Margin is the capital you put up to open a leveraged position. If the trade moves against you enough to exhaust your margin, the exchange "liquidates" the position — forcibly closes it, and you lose the margin.

The stop-loss prevents liquidation. Every trade has a stop-loss set well before the liquidation price. The bot's risk management ensures the SL triggers long before liquidation would occur.

Funding Rate

Perpetual futures use a "funding rate" mechanism. Every 8 hours, one side pays the other:

  • Positive funding: Longs pay shorts. Indicates the market is crowded long.
  • Negative funding: Shorts pay longs. Indicates the market is crowded short.

The bot monitors funding rates as part of its leverage analysis. Extreme funding suggests crowded positioning and increases the probability of a Judas swing (fake move to liquidate the crowded side).

What the Bot Monitors

The leverage analysis module tracks:

  • Open Interest (OI): Total value of outstanding futures contracts. Rising OI with price = real conviction. Rising OI against price = potential squeeze.
  • Long/Short Ratio: The balance between long and short positions. Extreme ratios indicate crowded trades.
  • Crowding Intensity: How extreme the positioning imbalance is (percentage).
  • Liquidation Levels: Estimated price levels where large clusters of long or short positions would be liquidated, creating liquidity for sweeps.

11 Bot Instances ▶

Tarakta can run multiple independent bot instances from the same codebase, each with its own identity, trade history, and configuration. This lets you test improvements without disrupting a running bot.

Active Instances

tarakta-expanded The original production bot. Uses the expanded-footprint scanner mode. Dashboard: tarakta-expanded.fly.dev
tarakta-stream Enhanced version with improved AI decision quality. Includes fixed RAG embeddings, lower agent temperatures for consistency, stricter entry gates, and sentiment-aware entry timing. Dashboard: tarakta-stream.fly.dev

How Isolation Works

Each instance has a unique INSTANCE_ID that keys its engine state, open positions, and configuration overrides in the shared database. Two bots can run the same code against the same exchange without interfering with each other. They share the same knowledge base (RAG) and trade lessons, so learnings from one bot benefit the other.

What Changed in tarakta-stream

  • RAG fix (critical): Embeddings were stored as JSON strings instead of pgvector arrays, completely breaking the dense search component of hybrid retrieval. Agents were seeing "No similar past trades found" on every signal. Now fixed — agents can actually learn from past trade outcomes.
  • Lower temperatures: Agent 1 (0.4), Agent 2 (0.5), Agent 3 (0.3) — down from 1.0. At temperature 1.0, the same signal could produce opposite decisions. Lower temperatures make decisions consistent and repeatable.
  • Must-reach-price gate enforced: Agent 2 now MUST wait for the institutional displacement move to occur before entering. Previously the gate was advisory ("note it but proceed"), allowing premature entries before the thesis was confirmed.
  • Sentiment passed to Agent 2: Agent 2 now sees the sentiment score (-10 to +10) and recent news headlines. Previously it was blind to news events that occurred between Agent 1's analysis and Agent 2's entry decision.
  • Lesson framing: Agents 2 and 3 now receive explicit "IMPORTANT: apply these lessons" instructions. Previously, lessons were appended as raw text at the end of the prompt with no emphasis — agents often ignored them.
  • Smarter lesson priority: Symbol-specific lessons matching the agent are shown first, then high-severity general lessons, then medium. Previously, generic high-severity lessons could fill all slots and crowd out relevant symbol-specific ones.
  • Richer RAG queries: Trade history retrieval now includes market regime (trending/ranging) and displacement context, not just the symbol and sweep type. Produces more relevant historical matches.
  • Better error logging: Advisor insight failures and RAG retrieval failures are now logged instead of silently swallowed, making it possible to diagnose issues.

Comparing Performance

Both bots share the same Supabase database but use separate instance IDs. You can compare their results by viewing each dashboard separately. The goal is to validate that tarakta-stream makes better decisions (higher win rate, better R:R, fewer premature entries) before promoting its changes.

12 Paper vs Live Trading ▶

Paper Trading

Uses real market data but all trades are simulated. No real money is at risk. The paper trading wrapper tracks positions, SL/TP levels, and PnL as if trades were real. Perfect for testing strategies, validating changes, and evaluating performance.

Live Trading

Real orders are placed on Binance Futures. Real money is used. Every profit and every loss is real. Slippage, fees, and funding payments apply.

Differences Between Paper and Live

  • Slippage: Paper trades execute at exact prices. Live trades may get slightly worse fills, especially on volatile pairs or large positions.
  • Fees: Paper mode may not perfectly model exchange fees. Live mode incurs maker/taker fees on every trade.
  • Funding: Live positions pay/receive funding every 8 hours. Paper mode simulates this.
  • Liquidity: Paper mode assumes infinite liquidity. Live mode may face partial fills on low-volume pairs.

How to Switch

Go to the Settings page and change the Trading Mode. This is an admin-only action.

Before Going Live

  • Run paper trading for at least 2–4 weeks with consistent profitability.
  • Understand every risk setting and what happens when it triggers.
  • Start with minimal leverage and small position sizes — you can always increase later.
  • Monitor the dashboard daily, especially in the first week. Check the Analytics page for trends.
  • Ensure the circuit breaker is set at a level you are comfortable with as a maximum loss.
  • Keep API key permissions minimal — the bot needs futures trading permission but not withdrawal permission.

Spotting Problems

  • Declining win rate over time: The market regime may have changed (trending to ranging) or the strategy needs recalibration.
  • Large individual losses: Check if SL settings are too wide or if the agents are placing stops poorly.
  • Many consecutive losses: Could indicate wrong market conditions for the strategy. The cooldown should kick in, but consider pausing manually.
  • High win rate but flat/negative PnL: Winners are too small relative to losers. The R:R ratio needs improvement — either tighter stops or wider take-profits.
  • API costs increasing without more trades: Agents may be retrying failed calls or the model is too expensive for the trade frequency.

What Success Looks Like

  • A steady upward P&L curve over weeks and months (not just a few good days).
  • Consistent R:R above 2:1 on average — winners significantly larger than losers.
  • Win rate holding above 50%, ideally 55–65%.
  • Drawdowns recovering relatively quickly (weeks, not months).
  • Lessons being generated and applied — the bot gets measurably smarter over time.
14 Glossary & Key Terms ▶
Absorption Large limit orders sitting at a price level, absorbing aggressive orders from the opposite side. Indicates institutional defence of a level.
BOS (Break of Structure) When price breaks a recent swing high (bullish BOS) or swing low (bearish BOS), confirming trend continuation.
CHoCH (Change of Character) When price breaks structure in the opposite direction of the trend, signalling a potential reversal.
Circuit Breaker Automatic safety switch that halts all trading when total losses exceed a set threshold. Prevents catastrophic drawdowns.
Confluence When multiple technical indicators agree on the same trade direction. Measured as a score out of 100. More confluence = higher confidence.
Cumulative Delta Running total of aggressive buy volume minus aggressive sell volume. Positive delta = net buying pressure. Negative = net selling.
Displacement A strong, high-volume price move away from a sweep level. Proves institutional commitment behind the reversal.
Drawdown The decline from a peak balance to a trough. A 10% drawdown on $10,000 means balance dropped to $9,000.
Entry Threshold Minimum confluence score a signal must reach to be considered. Higher threshold = fewer, higher-quality trades.
Fair Value Gap (FVG) A price imbalance where candle bodies don't overlap, leaving a gap. Price tends to return to fill these gaps.
Footprint Real-time order flow analysis. Examines the trade tape to see buyer/seller pressure, delta, absorption, and OI changes.
Funding Rate Periodic payment between long and short futures holders. Positive funding = longs pay shorts (market crowded long).
HTF (Higher Timeframe) Longer-period charts (4H, 1D) used to determine the larger trend. HTF alignment means the trade direction matches the bigger picture.
Judas Swing A fake move at the open of a session designed to trap traders before the real move. Probability is calculated from funding and OI data.
Kill Zone The first 1-2 hours of a major session (London/NY) when institutional manipulation (sweeps) is most likely.
Leverage Borrowing power that amplifies position size. 10x means $100 controls $1,000. Amplifies both gains and losses.
Liquidation Forced closure of a position when losses exhaust the margin. The bot's SL triggers well before this level.
Liquidity Sweep Price spikes past a key level to trigger stop-losses, then reverses. The primary signal in expanded-footprint mode.
Margin Your capital used to open a leveraged position. Your actual money at risk in a trade.
Market Structure The pattern of swing highs/lows defining a trend. Analysed on 1H, 4H, and 1D with strength percentages.
Open Interest (OI) Total value of outstanding futures contracts. Rising OI = new money entering. Falling OI = positions closing.
Order Block (OB) The last candle before a strong institutional move. Marks the zone where institutions placed orders. Acts as support/resistance.
OTE (Optimal Trade Entry) The 61.8%-78.6% Fibonacci retracement zone of a displacement move. The ideal pullback entry area.
P&L Profit and Loss. Net financial result of trading, measured in dollars.
Paper Trading Simulated trading using real market data but no real money. Used for testing and evaluation.
RAG Retrieval-Augmented Generation. The bot's knowledge system that retrieves similar past trades to inform current decisions.
R:R Ratio Reward-to-Risk ratio. A 3:1 R:R means potential profit is 3x the potential loss.
RVOL (Relative Volume) Current volume compared to the average for that time period. RVOL of 2.0 means volume is 2x normal.
SL (Stop Loss) Pre-set price where a trade closes to limit losses. Every trade must have one. Non-negotiable.
Smart Money Concepts (SMC) Trading methodology based on how banks and hedge funds operate. Focuses on sweeps, order blocks, FVGs, and displacement.
TP (Take Profit) Pre-set price where a trade closes to lock in profit. Tarakta uses three tiers: TP1 (quick lock-in), TP2 (medium), TP3 (runner).
