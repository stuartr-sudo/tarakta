Training Guide

A complete reference for understanding how Tarakta analyses markets, makes decisions, and manages risk.

01 System Overview ▶

Tarakta is an automated cryptocurrency futures trading bot that combines Smart Money Concepts (institutional trading strategies) with AI-powered decision making. It trades perpetual futures on Binance using leverage.

Core idea: The bot watches crypto markets around the clock, identifies trading opportunities that institutional traders would look for, then uses three AI agents to decide whether and when to trade. It can go both long (profit when price rises) and short (profit when price falls).

The Full Pipeline

Scanner → Confluence → Consensus → Agent 1 → Entry Refiner → Footprint → Agent 2 → Execute → Agent 3 → Lesson

How a Scan Cycle Works

Every scan interval (configurable, typically every few minutes), the engine:

  • Fetches 1H, 4H, and 1D candle data for all configured trading pairs
  • Runs market structure analysis across all timeframes to determine trend direction
  • Checks session timing (Asian, London, New York kill zones)
  • Runs sweep detection on the 1H timeframe — a completed liquidity sweep is the primary entry trigger
  • Confirms displacement (strong volume move away from the sweep level)
  • Scores the signal using the PostSweepEngine confluence formula
  • If score passes the entry threshold, enriches the signal with structural data and sends it to Agent 1

Scanner Mode

The active scanner mode strips out retail-level indicators (standalone OBs, FVGs, premium/discount zones) and focuses purely on sweep-first setups. A completed liquidity sweep on the 1H timeframe is the only entry trigger. OBs, FVGs, and liquidity levels are still computed but used only as context for the AI agents, not for scoring.

Instance Isolation

Multiple bot instances can run simultaneously against the same exchange and database. Each instance has a unique INSTANCE_ID that keeps its engine state, open positions, and configuration overrides separate. This allows A/B testing of different strategies or settings.

Infrastructure

  • Hosting: Fly.io, Amsterdam region (shared-cpu-1x, 2GB RAM)
  • Exchange: Binance Futures (perpetual contracts)
  • Database: Supabase (PostgreSQL with pgvector extension)
  • AI: OpenAI GPT models for agents, OpenAI embeddings for RAG
  • Dashboard: FastAPI + Jinja2 web interface on port 8080
02 The Three AI Agents ▶

Each agent is an OpenAI GPT model call with a detailed system prompt, structured JSON output, and rich context injected into the user prompt. They operate as a sequential chain — each agent's output feeds the next.

Agent 1 — Strategic Entry Analyst

Answers: "Is this trade worth taking, and in which direction?"

Agent 1 is the direction authority. It independently decides whether to go LONG or SHORT regardless of the scanner's initial suggestion. It returns one of three verdicts:

  • SETUP_CONFIRMED: Enter now — the signal is strong enough for immediate action.
  • WAIT_PULLBACK: The thesis is valid but the price is not yet at an optimal entry. The signal is queued in the Entry Refiner.
  • SKIP: Reject this signal entirely.

What Agent 1 Sees

Agent 1 receives an extensive context package including:

  • Sweep details (level, depth, type)
  • HTF market structure (1H/4H/1D trends)
  • 5 nearest order blocks with distance %
  • 5 nearest fair value gaps with midpoints
  • Liquidity pools (buy & sell side)
  • Relative volume (RVOL multiplier)
  • Displacement detection & strength
  • Pullback status & retracement %
  • Session timing (kill zone status)
  • Fibonacci retracement levels (OTE zone)
  • Leverage profile (funding, OI, L/S ratio)
  • Judas swing probability
  • Liquidation distances (long & short)
  • BTC trend & 24h change
  • Sentiment score (-10 to +10)
  • Recent news headlines
  • Last 5 trades on this symbol (outcome, PnL)
  • RAG knowledge (similar past trades)
  • Learned lessons from post-mortems
  • Advisor insights (missed opportunities)
  • Bot performance metrics (win rate, streaks)
  • Algorithmic SL/TP reference levels

Agent 1 outputs: action, direction (LONG/SHORT), confidence score, detailed reasoning, suggested entry zone, SL, TP, invalidation level, must-reach price, and market regime classification.

Agent 2 — Tactical Entry Timer

Answers: "Is NOW the right moment to enter?"

Agent 2 is called every 5 minutes while a signal is queued. It receives Agent 1's locked direction and thesis (cannot change it) plus fresh real-time data. It performs a mandatory 4-step analysis:

  • Step 0 — Must-reach-price gate: Has the institutional displacement move occurred?
  • Step 1 — Rejection check: Has price tested the entry zone and shown a rejection candle?
  • Step 2 — Live vs continuation: Is this a live rejection or stale data?
  • Step 3 — Structural confirmation: Do order blocks, FVGs, and market structure support entry?

What Agent 2 Sees (beyond Agent 1's context)

  • Last 50 five-minute candles (OHLCV table)
  • Order book (top 10 bid/ask, walls, imbalance)
  • Footprint / order flow (delta %, absorption)
  • Cumulative delta confirmation
  • OI change percentage
  • Buy/sell volume split
  • Previous Agent 2 analysis (reasoning continuity)
  • Check count (how many times evaluated)
  • Price change since queued
  • Sentiment score & headlines
  • Advisor insights

Agent 2 outputs: ENTER (with exact entry price, SL, TP, position size modifier 0.25–1.5x, confidence, urgency) or WAIT.

Agent 3 — Position Manager

Answers: "Should I adjust this open trade?"

Agent 3 monitors every open position and can do exactly two things:

  • TIGHTEN_SL: Move stop-loss closer to current price (never wider). Requires confidence ≥ 50.
  • EXTEND_TP3: Push the final take-profit target further. Requires confidence ≥ 60 and evidence of strong momentum.

Agent 3 defaults to HOLD in 80%+ of cases. It cannot close trades, widen stops, or override the exchange's hard SL/TP levels.

What Agent 3 Sees

  • Position details (entry, SL, TP, direction)
  • Current PnL ($ and %)
  • Unrealized R ratio
  • TP tier progress (X/3 filled)
  • Time held
  • Remaining position quantity
  • 1H ATR (volatility baseline)
  • Agent 1's original thesis
  • BTC trend + funding rate
  • HTF market structure per timeframe
  • Last 15-20 candles (5m + 1h)
  • Position management lessons

Temperature Settings

Temperature controls how random or deterministic the AI's output is. Lower = more consistent decisions. Agent 1: 0.4 (strategic but stable), Agent 2: 0.5 (some flexibility for timing), Agent 3: 0.3 (very deterministic, safety-critical).
03 Signal Scanning & Confluence Scoring ▶

The scanner checks all configured markets on a timer. The only entry trigger is a completed liquidity sweep on the 1H timeframe, confirmed by displacement volume.

Confluence Score Formula

Each signal is scored out of 100 base points (plus up to 10 bonus points for leverage alignment):

ComponentPointsWhat It Checks
Sweep Detected35 ptsA completed liquidity sweep on 1H — price spiked past a key level, triggered stops, and reversed.
Displacement Confirmed25 ptsStrong volume move away from the sweep level. Proves institutional commitment.
HTF Direction Aligned15 ptsThe 4H and/or 1D market structure trend agrees with the trade direction.
Post-Kill-Zone Timing15 ptsSignal occurs shortly after an active session kill zone (London or New York open).
Pullback Confirmed10 ptsPrice has pulled back into the OTE zone (61.8%–78.6% Fibonacci retracement).
Leverage Aligned (bonus)0–10 ptsFunding rate, open interest, and long/short ratio all support the direction.
Entry Threshold: Signals must reach the configured minimum score (set in Settings) to be passed to Agent 1. A typical threshold is 60–70.

Weekly Cycle Adjustments

  • Monday manipulation window: Score penalty during early Monday hours when institutions often create fake moves.
  • Mid-week reversal bonus: Small score boost during Tuesday–Thursday when genuine trend moves are more likely.

Consensus Check

Before reaching Agent 1, signals pass through a consensus mechanism:

  • Portfolio bias: If most open profitable positions are in the opposite direction, a penalty of −10 points is applied.
  • BTC trend disagreement: If BTC's 4H/1D structure contradicts the signal direction, −5 points.

If a signal's score drops below threshold after penalties, it is queued in the Consensus Monitor and re-evaluated every 60 seconds for up to 30 minutes.

Session Timing & Kill Zones

  • Asian session (00:00–08:00 UTC): Lower volume. Price often establishes a range.
  • London session (07:00–16:00 UTC): High volume. Institutions often sweep Asian session highs/lows.
  • New York session (12:00–21:00 UTC): Highest volume. Most significant moves occur here.

Footprint Gate

After Agent 2 says ENTER, the Footprint Analyzer examines real-time order flow to confirm buyer/seller pressure supports the direction:

  • Delta %: Ratio of aggressive buys to sells. For longs, delta should be positive.
  • Absorption: Large buy orders sitting at a level absorbing sell pressure.
  • Cumulative delta: Running total confirming directional commitment.
Note: Footprint data is ephemeral — it comes from the real-time trade tape and cannot be fetched historically. If the footprint gate fails, the trade is rejected.

Emergency brake. Automatically halts ALL trading if total losses exceed a threshold. When triggered, no new trades are placed until manually reset. Prevents catastrophic losses during black swan events.

  • Daily Drawdown Limit: Pauses trading until the next day if daily losses exceed the configured limit.
  • Cooldown Period: After consecutive losses, no new trades for a configurable period.
  • BTC Macro Gate: Blocks altcoin longs when BTC is crashing.
  • Sentiment Gate: Extreme negative sentiment can block or reduce scores for long trades.
  • Funding Rate Gate: Blocks trades against heavily funded positions.
05 Position Management & Trade Lifecycle ▶

Full Trade Lifecycle

Signal scored → Agent 1 approves → Queued → Agent 2 enters → Footprint gate → Order placed → Agent 3 manages → Trade closes → Lesson

Entry Refiner & Pullback Queue

When Agent 1 says WAIT_PULLBACK, the signal enters the Entry Refiner queue:

  • Entry zone: Defined by Agent 1, typically based on an order block, FVG, or Fibonacci retracement zone.
  • OTE zone: The 61.8%–78.6% Fibonacci retracement of the displacement move.
  • Must-reach price: A price level that must be hit before pullback entry is valid.
  • Agent 2 checks: Every 5 minutes with fresh 5m candles, order book, and footprint data.
  • Expiry: Signals expire after a configurable period if Agent 2 never says ENTER.

Take-Profit Tiers

TierPurposeBehaviour
TP1Quick lock-inCloses a portion early, reducing risk by banking profit immediately.
TP2Medium targetAnother portion at a higher target. SL typically moved to breakeven.
TP3Full target / runnerRemaining position. Can be extended by Agent 3 if momentum is strong.

Trailing Stop & Breakeven

  • Breakeven activation: After a configurable R:R ratio, SL moves to entry price. Trade cannot lose money from this point.
  • Trailing activation: At a higher R:R threshold, a trailing stop follows price, locking in progressively more profit.

How Trades Close

  • Stop-loss hit: Closes at a loss (or breakeven/profit if SL was tightened).
  • All TP tiers filled: Full profit captured across all three levels.
  • Max hold time exceeded: Configurable time limit prevents capital from being tied up.
  • Stale close: Low R:R after extended hold time.

Post-Trade: Lesson Generation

After every trade closes, the Lesson Generator analyses the outcome. It produces structured lessons tagged with severity, type, and which agents should learn from it. These are stored in the database and injected into future agent prompts.

06 AI Learning & Knowledge Systems ▶

Tarakta has three systems that make agents smarter over time: the RAG knowledge base, the lesson generator, and the trade advisor.

RAG (Retrieval-Augmented Generation)

Purpose: Before making a decision, agents search past trade history for similar setups and see how those trades turned out.
  • Ingestion: Every closed trade is embedded as a 1536-dimensional vector using OpenAI's embedding model.
  • Storage: Embeddings stored in Supabase using pgvector alongside text content.
  • Retrieval: Hybrid search combines dense (cosine similarity) with lexical (full-text) using Reciprocal Rank Fusion.
  • Injection: Top 5 most relevant past trades are formatted and injected into Agent 1 and Agent 2's context.

Lesson Generator

Purpose: Extract actionable rules from every closed trade so agents learn from mistakes and successes.
  • Reviews all three agents' reasoning, the trade outcome, and signal quality
  • Generates structured lessons in "IF [condition], THEN [action]" format
  • Tags each lesson with severity, type, and which agents it applies to
  • Lessons are injected into relevant agent prompts with priority ordering

Trade Advisor

Purpose: Identify missed opportunities and patterns the bot is consistently missing.
  • Fetches high-scoring signals that were never traded
  • Simulates what would have happened if the trade had been taken
  • Uses Claude to analyse patterns across missed signals
  • Produces recommendations stored in the advisor_insights table
  • Insights are injected into Agent 1 and Agent 2's context
07 Smart Money Concepts Explained ▶

Smart Money Concepts (SMC) is a trading methodology based on understanding how large institutional traders operate. These institutions control the majority of market volume and create predictable patterns.

Liquidity Sweeps (Primary Signal)

The core thesis: Institutional traders need liquidity to fill their large positions. They deliberately push price to levels where retail traders have stop-losses, trigger those stops, then reverse the market in their intended direction.

A liquidity sweep occurs when price briefly spikes above resistance or below support, triggers stop-losses, then reverses. A confirmed sweep on the 1H timeframe is the ONLY trigger for a new signal.

Displacement

After a sweep, institutions make a strong move in their intended direction. This "displacement" is characterised by large-body candles with high volume. The scanner awards 25 points for confirmed displacement.

Fair Value Gaps (FVGs)

When price moves so fast that candle bodies don't overlap, it creates an imbalance zone. Price tends to return to fill these gaps. FVGs are used as potential entry zones.

Order Blocks (OBs)

The last candle before a strong institutional move. Represents the zone where institutions placed their orders. Acts as support/resistance when price returns.

Market Structure

  • Break of Structure (BOS): Price breaks a recent swing high/low, confirming trend continuation.
  • Change of Character (CHoCH): Price breaks structure in the opposite direction, signalling potential reversal.

Analysed on three timeframes (1H, 4H, 1D) with strength percentages. HTF alignment adds 15 points to confluence.

Optimal Trade Entry (OTE)

The 61.8%–78.6% Fibonacci retracement of a displacement move. The statistically optimal area where price tends to pull back before continuing.

Judas Swing

A fake move at session open designed to trap traders on the wrong side before the real move. Probability calculated from funding rates, open interest, and market positioning.

08 Leverage, Futures & Margin ▶

Tarakta trades perpetual futures contracts on Binance. Understanding leverage and margin is critical.

Perpetual Futures

  • Long: Profit when price goes up, lose when price goes down.
  • Short: Profit when price goes down, lose when price goes up.

"Perpetual" means the contract never expires. You can hold as long as you maintain margin.

Leverage

Leverage amplifies your position size. At 10x leverage, $100 of margin controls $1,000 worth of a position. A 1% move in your favour = 10% on your margin.

Margin & Liquidation

Margin is the capital you put up to open a leveraged position. If losses exhaust your margin, the exchange "liquidates" the position.

The stop-loss prevents liquidation. Every trade has a stop-loss set well before the liquidation price. Risk management ensures the SL triggers long before liquidation would occur.

Funding Rate

Every 8 hours, one side pays the other:

  • Positive funding: Longs pay shorts (market crowded long).
  • Negative funding: Shorts pay longs (market crowded short).

What the Bot Monitors

  • Open Interest (OI): Total outstanding futures contracts. Rising OI with price = conviction.
  • Long/Short Ratio: Balance between long and short positions.
  • Crowding Intensity: How extreme the positioning imbalance is.
  • Liquidation Levels: Price levels where large clusters of positions would be liquidated.

12 Paper vs Live Trading ▶

Paper Trading

Uses real market data but all trades are simulated. No real money at risk. Perfect for testing strategies and evaluating performance.

Live Trading

Real orders on Binance Futures. Real money. Slippage, fees, and funding payments apply.

Key Differences

  • Slippage: Paper executes at exact prices. Live may get worse fills.
  • Fees: Paper may not perfectly model exchange fees.
  • Funding: Live positions pay/receive funding every 8 hours.
  • Liquidity: Paper assumes infinite liquidity. Live may face partial fills.

Before Going Live

  • Run paper trading for at least 2–4 weeks with consistent profitability
  • Understand every risk setting and what happens when it triggers
  • Start with minimal leverage and small position sizes
  • Monitor the dashboard daily, especially in the first week
  • Ensure the circuit breaker is set at a level you're comfortable with
  • Keep API key permissions minimal — no withdrawal permission

15 MM Method — Core Philosophy ▶

The Market Makers Method is built on the premise that price is not random — it follows a repeatable business model operated by institutional Market Makers (Citadel, Jump Crypto, Virtue Financial, Cumberland) who work in 6–8 hour shifts across three global sessions.

Core idea: MMs MUST hit stop losses and liquidation levels to fill their positions. They can't use limit orders (visible) or market orders (moves price). So they engineer traps — and those traps create predictable patterns we can trade.

Key Axioms

  • Slow price action = REAL move. Fast price action = FALSE move (trap). This is counter-intuitive but fundamental.
  • MMs have deadlines: Weekly, monthly, and seasonal targets set by IMF/World Bank. The target must be met by Friday 5pm NY.
  • The MM can do anything midweek as long as the target is met by close.
  • Same pattern, every timeframe: Smaller patterns build into larger ones (fractal).
  • Only 3 wins out of 10 needed at 4:1 R:R to be profitable.

Expected Performance

Risk-to-Reward4:1 to 10:1 (up to 20:1)
Min win rate for profit30% at 4:1 R:R

How This Differs from SMC

  • SMC (sections 01–14): Sweep-first triggers, AI agents decide entries, real-time footprint gates
  • MM Method (sections 15–33): Weekly cycle template, M/W formations at session changeovers, 3-level swing targets, zero LLM calls, fully algorithmic

Both run simultaneously. The MM Engine is a standalone asyncio task alongside the SMC engines.

16 The Weekly Cycle Template ▶

The MM operates on a predictable weekly business model. Every week follows the same sequence:

Weekend Trap → FMWB → M or W → 3-Level Swing → Midweek Rev. → Opposite Swing → Friday Trap

Sequence Breakdown

  • Weekend Trap (Fri 5pm → Sun 5pm NY): Sideways consolidation, spike stop-hunt near end
  • False Move Week Beginning (FMWB): Aggressive trap move at/near Sunday 5pm NY (crypto) or UK/US open Monday (forex). Purpose: stop out weekend traders, build MM positions
  • M or W Formation: Forms after FMWB — entry on 2nd peak
  • 3-Level Swing (~3 days): Level 1 → Board Meeting → Level 2 → Board Meeting → Level 3
  • Midweek Reversal (Wed or Thu for crypto, Wed for forex): Lines up with a news event. M or W forms at Level 3, signals direction change
  • 3-Level Swing in opposite direction
  • Friday Weekend Trap (UK session): UK opens with false move, runs trend 4–6 hours, repeats initial level by end of UK session
Key insight: The week begins and ends at 5pm New York time. Every day also starts at 5pm NY. Not midnight, not UTC — always 5pm NY.

Implementation

18 The Three Elements ▶

Every MM Method trade requires all three elements to align. Missing any one of them significantly reduces probability.

Element 1 — The Pattern

  • Trap moves (FMWB) → clear weekend leverage, build contracts
  • M or W formation → M = bearish (High → Lower High), W = bullish (Low → Higher Low)
  • Three Levels of rises (after W) or drops (after M)
  • Board Meetings between levels = consolidation where MM accumulates contracts
  • Wick behaviour shift at Level 3: wicks move from one side to the other → reversal hint
  • Stopping Volume Candle (SVC) at Level 3 confirms trend exhaustion
  • New M or W at the other extreme → entry for the next swing

Element 2 — The Timing

  • M/W must form at session changeovers (Gap times)
  • Multi-session M/W (peaks in different sessions) = strongest setup
  • Midweek reversal: Wed/Thu (crypto), Wed (forex)
  • Friday UK session: take profit day — exit during peak formation

Element 3 — The Levels

  • Levels = bursts/fast movements in price, identified by abnormally high volume
  • Always count to 3 levels. A 4th level (Extended Rise/Drop) almost always brings correction
  • Previous week's High/Low and previous day's High/Low replace traditional S/R
  • Only care about positions opened within the last 3–5 days

19 M/W Formations ▶

M/W formations are the primary entry signal. An M formation is bearish (price topped), a W formation is bullish (price bottomed). Four variants exist.

Standard M/W — Three MM Appearances

  • 1. Stopping Volume Candle (SVC): Small body, large wick in trend direction, very high volume at Level 3. Mark the zone — price should not fully return.
  • 2. Inside Right Side (drop one timeframe): Look for 3 trap candles (3 green for M, 3 red for W).
  • 3. 50 EMA break with volume: Final confirmation.
Key rules: The 2nd peak must NOT reach the 1st peak (MM keeps traders trapped). Spikes must pull away quickly. Wait for candle CLOSE as hammer/engulfing before entering. M/W is NOT confirmed until it successfully retests.

Multi-Session M/W (Most Powerful Setup)

  • 1st peak in one session, 2nd peak in another (e.g., Asia peak → UK open peak)
  • Found at HOW/LOW = weekly reversal signal with tight SL and massive R:R

Final Damage M/W

  • 2nd peak makes a LOWER low (W) or HIGHER high (M) than 1st peak
  • Must be a hammer (W) or inverted hammer (M) on 15min
  • Better R:R but higher probability of being wrong

Board Meeting M/W

  • Forms inside consolidation between levels
  • Does NOT need full standard criteria (no SVC, no 3 inside hits)
  • Just the shape is sufficient

Three Hits Rule

  • 3 tests of HOW or LOW without breaking → reversal imminent (must be at Level 3)
  • 4 hits → continuation/breakout likely
  • Hits must be in DIFFERENT sessions (up to 2 per session)
  • After each hit, price must move away then return

20 EMA Framework (5 EMAs) ▶

Five EMAs used on ALL timeframes. They serve as dynamic support/resistance and define the level progression.

EMA Cycle Through Levels

  • Entry: Price crosses 10 and 20 EMA (usually on 2nd M/W peak)
  • Level 1: Breaks 50 EMA with volume (MUST have volume)
  • Board Meeting: Retraces to 50 EMA (if only to 10 EMA and counter to macro trend → caution)
  • Level 2: Runs to 200 EMA. Rejection at 200 EMA with hammer = TP zone
  • Level 2 Board Meeting: Retests 50 EMA
  • Level 3: Trend acceleration — EMAs fan out, price extends far from EMAs. This traps retail.

Trend Ending Signals

  • EMAs flatten (stop pointing in trend direction)
  • Stopping Volume Candle appears
  • Gap between price and EMAs = magnetic attraction → reversion coming

21 Entry Rules ▶

Two Entry Types

Aggressive Entry (on 2nd peak of M/W): Enter when candle CLOSES as hammer/engulfing on 2nd peak. SL above 1st peak wick (M) or below 1st peak wick (W). Typical R:R: 4:1 to 7.8:1.
Conservative Entry (after Level 1 confirmation): Wait for 50 EMA break with volume. Enter on retest of the 50 EMA or board meeting entry. SL above 2nd peak of M/W. Typical R:R: 2.7:1 to 9.8:1.

Four Retest Conditions (need ≥2 for entry)

After Level 1, price must retrace to a combination of at least 2:

  • The 50 EMA
  • The vector that created the Level 1 move
  • A higher low (after W) or lower high (after M)
  • A heat map / liquidation level cluster

Board Meeting Entries

  • Retracement Board Meeting: Fibonacci from peak to trough — stagger orders at 38.2%, 50%, 61.8%. SL above the peak of prior level.
  • Sideways Board Meeting: Look for M/W shape inside consolidation (reduced criteria). Stop hunt comes at END of board meeting, not beginning.

When NOT to Enter

  • R:R below 1.4:1
  • Friday after 5pm NY market close
  • Setup doesn't show clear M/W and clear 3 levels → skip
  • After only Level 1 drop — do NOT counter-trend trade from 200 EMA back
  • Asia range >2% for BTC → skip the day

26 Weekend Trap & FMWB ▶

The Weekend Trap

Every week begins and ends with a trap. The MM uses the low-volume period to engineer sideways ranges that lure traders into positions that will be stopped out.

  • Duration: Friday 5pm NY → Sunday 5pm NY
  • Character: Sideways consolidation with a spike stop-hunt near the end
  • Purpose: Trap weekend traders who take positions based on the range

Weekend Trap Box Marking (1H)

  • Mark from US Friday close candle
  • Everything from next candle through to Dead Gap Zone (5pm NY Sunday) = the box
  • Mark candle closes (not wicks) above and below within the box
  • Identify who got trapped (shorts induced early → stopped out late, etc.)

False Move Week Beginning (FMWB)

The most important move of the week: An aggressive trap move at Sunday 5pm NY (crypto) or UK/US open Monday (forex). It stops out weekend traders and builds MM positions for the real weekly move.
  • Happens fast — remember: fast moves = false moves
  • Direction is OPPOSITE to the real weekly trend
  • After FMWB completes, look for M/W formation

Friday Weekend Trap

  • UK session opens with a false move
  • Runs the trend for 4–6 hours
  • Repeats the initial level by end of UK session — this IS the trap
  • Friday is a take-profit day, not a new-entry day

27 Board Meeting Entries ▶

Board Meetings are consolidation zones between levels where the MM accumulates more contracts. They provide some of the best re-entry points.

Retracement Board Meeting

  • Use Fibonacci from peak to trough
  • Stagger orders at 38.2%, 50%, 61.8%
  • SL above the peak of prior level
  • Price often retraces to the 50 EMA during board meetings

Sideways Board Meeting

  • Look for M/W shape inside the consolidation (reduced criteria — no SVC needed)
  • Stop hunt comes at END of board meeting, not beginning
  • The shape alone is sufficient for entry
Board Meeting timing: If the retracement only reaches the 10 EMA (not the 50) and it's counter to the macro trend, exercise caution — the board meeting may be too shallow for a high-probability re-entry.

28 Risk Management (MM Method) ▶

Position Sizing

  • Risk per trade: 1% of total account (all exchanges combined)
  • Formula: Position Size = Risk Amount / Stop Loss Distance %
  • Example: $10K account, 1% risk = $100. SL at 1.33% distance → Position = $100 / 0.0133 = $7,519

Leverage

Leverage is a capital-freeing tool, NOT a profit multiplier. Dollar profit is IDENTICAL regardless of leverage (only ROI% changes). You probably don't need past 10x even with 2–3 simultaneous trades. Higher leverage = closer liquidation + higher fees.

  • Never tighten SL to improve R:R

  • Same R:R produces same dollar profit on any timeframe

29 Multi-Timeframe Charting Process ▶

Daily process (~15–20 minutes). Always chart levels FIRST, then check liquidation/order flow data. Never look at Hyblock/TradingLite before marking your own levels.

Monthly → Weekly → Daily → 12H/8H/6H → 4H → 1H → 15min

Timeframe Roles

  • Monthly (check end/start of month): Mark unrecovered Vectors above/below price as Areas of Interest
  • Weekly (check end/start of week): Refine areas, mark 50% lines of Vectors, build bull AND bear case
  • Daily (check every day): Refine areas further, mark levels, build both cases
  • 12H/8H/6H: Quick scan only — is the macro trend changing?
  • 4H: Check areas, vectors, heat map orders, calculate R:R for potential trades
  • 1H: Mark vectors, Weekend Trap Box, FMWB, Peak Formation, level count, SVCs. This is the primary trading timeframe.
  • 15min: Drop to this ONLY when entry is imminent on 1H, then return to 1H

Weekly Swing Checklist (1H)

  • Weekend Trap Box → identify fake move
  • Peak Formation → start level count
  • Count complete? → Look for reversal (M/W)
  • Count in progress? → Identify Level 3 target
  • If M/W formed: Did it break 50 EMA with volume? → Level 1 confirmed
  • Plan ahead: reversal location, news event, liquidation levels, heat map orders
  • Moon cycles: check full/new moon positions (14 days apart, ±3 day buffer)
  • News events: check 1–2 days ahead (red/orange impact, USD filter)
